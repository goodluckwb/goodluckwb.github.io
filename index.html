第一次Git错误的解决方法

2018/7/22 author

Step1.直接在你git的命令行窗口输入： remote remove origin.

Step2.把.ssh文件删掉，就是命令行ssh-add -D然后继续rm -r ~/.ssh就OK了。

Step3.重建ssh，直接输入命令ssh-keygen -t rsa -C “邮箱”,一路回车。 这里注意ssh-keygen命令中间没有空格，如果在ssh后面加上空格， 会得到Bad escape character ‘ygen’.的错误。

Step4.删除github网页里的ssh，在setting下的SSH and GPG keys下删除ssh。 输入新的ssh，新建一个key值，标题随意，key就是打开那个id_rsa.pub文件， 然后全部复制粘贴到key即可.

测试ssh -T git@github.com。

Step5.echo "new Test" >> README.md

git init

git add README.md

git commit -m "first commit"

git remote add origin 你的ssh地址

git push -u origin master
